Apps

Applications, or Apps, are objects that define a remote application that is authorized to use the Quartino web services.

Only properly authenticated administrators can add, remove, or list applications.

Listing Apps

Example Request

# Note: These examples use a QuartinoRestClient object which has not yet been
#       released as a separate library. If you desire the implementation for
#       this, please email Scott at jaywir3@gmail.com.
rest_client = QuartinoRestClient.new
response = rest_client.list_apps(@authenticated_user)

curl https://api.qr-invited-com/apps \
  -H "Authorization: Basic aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa" \
  -H "X-UserId: 888"

List all apps in the system.

This endpoint is not available to all users of the system. Only authenticated administrators are allowed to list apps in the system.

HTTP Request

GET https://api.qr-invited-com/apps

Request Headers

Header Value Description
Authorization Bearer auth_token Authorization token obtained via authentication mechanism (described above)
X-UserId user_id User id of authenticated user. This user must have the role of admin, or a 403 Forbidden will be sent back by the server.

Request Parameters

No parameters are required for this API call.

Get a Specific App

Example Request

# Note: These examples use a QuartinoRestClient object which has not yet been
#       released as a separate library. If you desire the implementation for
#       this, please email Scott at jaywir3@gmail.com.
rest_client = QuartinoRestClient.new
id = 9082
response = rest_client.get_app(@authenticated_user, id)

curl https://api.qr-invited-com/apps/9082 \
  -H "Authorization: Basic aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa" \
  -H "X-UserId: 888"

Example Response

{
  "id": 9082,
  "app_name": "My Quartino App",
  "requester_email": "some.user@qr-invited.com",
  "api_key": "30ef8ba034adeb693899b65d74e169dba40f3211"
}

Retrieves a specific app, by that app's id.

If the user authenticated is not an admin, then it's possible that a 403 Forbidden will be returned if the authenticated user is not the owner of the requested app.

HTTP Request

GET https://api.qr-invited-com/apps/<ID>

Request Headers

Header Value Description
Authorization Bearer auth_token Authorization token obtained via authentication mechanism (described above)
X-UserId user_id User id of authenticated user

URL Parameters

Parameter Description
ID The ID of the app to retrieve
