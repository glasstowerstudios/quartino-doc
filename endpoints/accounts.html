Accounts

The Account object contains the data related to a particular client. Quartino does not store any data regarding client billing methods (e.g. credit card, checking account, etc...), but it does store logistical information about client organizations and contact information. An account is a representation of a client record within Quartino, and contains basic information about who is authorized to make changes to an instance of QR-Invited, the subdomain used to access the site for a particular client, and other information related to the client's ability to use the QR-Invited services.

Retrieve all users in the system, without authentication information.

This is a listing of all users, but it does not include authentication information (e.g. passwords, password digests, authentication tokens, etc...).

This endpoint is available to all authenticated users of the system, but non-admins will only be able to view users within their respective `Account`(s).

HTTP Request

GET https://api.qr-invited-com/users

Request Headers

Header Value Description
Authorization Bearer auth_token Authorizatioqqwn token obtained via authentication mechanism (described above)
X-UserId user_id User id of authenticated user

Request Parameters

No parameters are required for this API call.
