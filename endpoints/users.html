Users

The User object describes an individual who can login to the QR-Invited system. It may be an account owner, a global administrator, or someone who has access to view the information on behalf of a particular client.

Guests invited to an event through the QR-Invited system are not users.

Get All Users

Example Request

# Note: These examples use a QuartinoRestClient object which has not yet been
#       released as a separate library. If you desire the implementation for
#       this, please email Scott at jaywir3@gmail.com.
rest_client = QuartinoRestClient.new
response = rest_client.list_users(@authenticated_user)

curl https://api.qr-invited-com/users \
  -H "Authorization: Basic aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa" \
  -H "X-UserId: 888"

Example Response

[
  {
    "id": 888,
    "name": "Jimmy Buffett",
    "username": "jbuffett",
    "email": "jimmy.buffett@margaritaville.com",
    "role": "admin"
  },
  {
    "id": 1234,
    "name": "Michael Utley",
    "username": "misterutley01",
    "email": "michael.utley@margaritaville.com",
    "role": "user"
  }
]

Retrieve all users in the system, without authentication information.

This is a listing of all users, but it does not include authentication information (e.g. passwords, password digests, authentication tokens, etc...).

This endpoint is available to all authenticated users of the system, but non-admins will only be able to view users within their respective `Account`(s).

HTTP Request

GET https://api.qr-invited-com/users

Request Headers

Header Value Description
Authorization Bearer auth_token Authorizatioqqwn token obtained via authentication mechanism (described above)
X-UserId user_id User id of authenticated user

Request Parameters

No parameters are required for this API call.

Get a Specific User

Example Request

# Note: These examples use a QuartinoRestClient object which has not yet been
#       released as a separate library. If you desire the implementation for
#       this, please email Scott at jaywir3@gmail.com.
rest_client = QuartinoRestClient.new
id = 1234
response = rest_client.get_user(@authenticated_user, id)

curl https://api.qr-invited-com/users/1234 \
  -H "Authorization: Basic aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa" \
  -H "X-UserId: 888"

Example Response

{
  "id": 1234,
  "name": "Michael Utley",
  "username": "misterutley01",
  "email": "michael.utley@margaritaville.com",
  "role": "user"
}

Retrieves a specific user, by that user's id.

If the authenticated user is not an admin, then it's possible that a 403 Forbidden will be returned if the id given corresponds to a user that is not associated with one of the Account objects for which the authenticated user is an administrator.

HTTP Request

GET https://api.qr-invited-com/users/<ID>

Request Headers

Header Value Description
Authorization Bearer auth_token Authorization token obtained via authentication mechanism (described above)
X-UserId user_id User id of authenticated user

URL Parameters

Parameter Description
ID The ID of the user to retrieve
