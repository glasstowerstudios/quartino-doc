Introduction

Welcome to Quartino! This is the web service for QR-Invited. Before working with the Quartino API, you'll need to register and get an application/api key from our administrators. This key serves to identify you as a developer within our systems, and to track usage of our API by application.

Authentication

Most of the API requires that a user be authenticated prior to making subsequent calls. Any call requiring an authorization token (auth token) requires that the authentication procedure be completed prior to that call. Authenticated API calls require an auth token and user id, which are obtained from the following endpoint:

Login with Username and Password in Body

Example Request

# Note: These examples use a QuartinoRestClient object which has not yet been
#       released as a separate library. If you desire the implementation for
#       this, please email Scott at jaywir3@gmail.com.
rest_client = QuartinoRestClient.new
response = rest_client.login username, password

curl -X POST "https://api.qr-invited-com/login" --data '{"username": "jbuffett", "password": "abcdef"}' \
  -H "X-Api-Key: mydeveloperkey"

Example Response


{
  "id": 888,
  "name": "Jimmy Buffett",
  "username": "jbuffett",
  "email": "jimmy.buffett@margaritaville.com",
  "role": 1,
  "auth_token": "aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa",
  "auth_token_expire_date": "2016-04-18T14:46:27.000Z"
}

This endpoint retrieves user data (including auth_token) for a specific user.

HTTP Request

POST /login

Request Headers

Header Value Description
X-Api-Key api_key API key for your application

Request Parameters

Parameter Default Description
username none The username of the user to authenticate with
password none The password of the user to authenticate with

These parameters can be combined and sent as an authorization header using Base64-encoding. See Login with Authorization Header for more information.

HTTP Request

POST /login

Request Headers

Header Value Description
X-Api-Key api_key API key for your application
Authorization Basic base64(username + ":" + password) Base64-encoded username and password

Request Parameters

No parameters are required for this API method.

This login can also be sent in separate username/password format in the body. See Login with Username and Password in Body for more information

Users

Get All Users

Example Request

# Note: These examples use a QuartinoRestClient object which has not yet been
#       released as a separate library. If you desire the implementation for
#       this, please email Scott at jaywir3@gmail.com.
rest_client = QuartinoRestClient.new
response = rest_client.list_users(@authenticated_user)

curl https://api.qr-invited-com/users \
  -H "Authorization: Basic aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa" \
  -H "X-UserId: 888"

Example Response

[
  {
    "id": 888,
    "name": "Jimmy Buffett",
    "username": "jbuffett",
    "email": "jimmy.buffett@margaritaville.com",
    "role": "admin"
  },
  {
    "id": 1234,
    "name": "Michael Utley",
    "username": "misterutley01",
    "email": "michael.utley@margaritaville.com",
    "role": "user"
  }
]

Retrieve all users in the system, without authentication information.

This is a listing of all users, but it does not include authentication information (e.g. passwords, password digests, authentication tokens, etc...).

This endpoint is available to all authenticated users of the system, but non-admins will only be able to view users within their respective Account(s).

HTTP Request

GET https://api.qr-invited-com/users

Request Headers

Header Value Description
Authorization Bearer auth_token Authorization token obtained via authentication mechanism (described above)
X-UserId user_id User id of authenticated user

Request Parameters

No parameters are required for this API call.

Get a Specific User

Example Request

# Note: These examples use a QuartinoRestClient object which has not yet been
#       released as a separate library. If you desire the implementation for
#       this, please email Scott at jaywir3@gmail.com.
rest_client = QuartinoRestClient.new
id = 1234
response = rest_client.get_user(@authenticated_user, id)

curl https://api.qr-invited-com/users/1234 \
  -H "Authorization: Basic aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa" \
  -H "X-UserId: 888"

Example Response

{
  "id": 1234,
  "name": "Michael Utley",
  "username": "misterutley01",
  "email": "michael.utley@margaritaville.com",
  "role": "user"
}

Retrieves a specific user, by that user's id.

If the user authenticated is not an admin, then it's possible that a 403 Forbidden will be returned if that user is not associated with the account for which the requested user is associated.

HTTP Request

GET https://api.qr-invited-com/users/<ID>

Request Headers

Header Value Description
Authorization Bearer auth_token Authorization token obtained via authentication mechanism (described above)
X-UserId user_id User id of authenticated user

URL Parameters

Parameter Description
ID The ID of the user to retrieve

Errors

The Quartino API uses the following error codes:

Error Code Meaning
400 Bad Request -- The specified request is incorrect in some manner.
401 Unauthorized -- Your API key is incorrect.
403 Forbidden -- The authenticated user does not have access to this resource.
404 Not Found -- The specified resource could not be found.
405 Method Not Allowed -- You tried to access a kitten with an invalid method
406 Not Acceptable -- A format that isn't json was requested.
410 Gone -- The requested resource has been removed from our systems.
429 Too Many Requests -- The server has detected too many requests in too short a period of time. Try slowing down your requests.
500 Internal Server Error -- There was a problem in the API itself. Likely this is something that should be filed as a bug.
503 Service Unavailable -- The Quartino API is undergoing maintenance and will be down for a period of time.
